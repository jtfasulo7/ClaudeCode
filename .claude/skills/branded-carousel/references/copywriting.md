# Carousel Copywriting

Reference loaded when drafting copy for slides. The goal of every slide is to earn the next swipe; the goal of the last slide is to convert.

## Cover slide (the hook)

The cover competes against everything in the feed for one second of attention. It must do three things at once: stop the scroll, name the audience, and promise a payoff.

### Hook formulas (use one)

1. **The contrarian** — "Stop [common practice]. Here's what to do instead."
   - *Stop posting daily. Here's the cadence that actually grows accounts.*
2. **The numbered list** — "[N] [things] that [outcome]."
   - *7 prompts that 10x your Claude output.*
3. **The mistake** — "The [N] [topic] mistakes costing you [outcome]."
   - *The 3 carousel mistakes costing you saves.*
4. **The question** — "Why [unexpected fact]?" or "What if [premise]?"
   - *Why does your best post get the worst reach?*
5. **The before/after** — "From [bad state] to [good state] in [time]."
   - *From 200 to 12k followers in 90 days.*
6. **The reveal** — "I [did unusual thing]. Here's what happened."
   - *I posted nothing for 30 days. Engagement went up.*
7. **The framework** — "The [name] framework for [outcome]."
   - *The PEEL framework for carousel copy that converts.*

### Cover length rule

- Headline: ≤ 8 words. Anything longer fights the type size.
- Subhead (optional): ≤ 12 words, sets the scope. Use only if the headline is ambiguous.
- No body copy on the cover. The cover is a billboard, not a paragraph.

## Context slide (slide 2)

One sentence. Establish the stake. Why does this matter *now*, to *this* reader?

Patterns:
- *Most creators do X. The result: Y.* (sets the problem)
- *Here's what nobody tells you about X.* (sets the secret)
- *If your X looks like Y, this is for you.* (sets the audience)

If the user asks for a 5-slide carousel, fold context into the cover and skip this slide.

## Value slides (slides 3 to N-1)

One idea per slide. The structure inside each value slide is **claim → reason → proof**.

### Layout pattern (text-only value slide)

```
[Big number or short headline — 4 to 8 words]

[Body — 1 to 3 sentences, max 35 words total]

[Optional: 1-line example, italicized]
```

### Layout pattern (stat slide)

```
[Stat in display font, 240pt, teal or white]
   ↑
[12-word framing line below]
```

### What to cut

- Adverbs ("really", "very", "literally")
- Hedge words ("maybe", "perhaps", "I think")
- Throat-clearing ("So basically", "Here's the thing")
- Any sentence that doesn't earn its slide

### Word budget per value slide

- Headline: ≤ 8 words
- Body: ≤ 35 words
- Total: ≤ 50 words. If a slide exceeds 50 words, split or cut.

## CTA slide (last slide)

The biggest mistake: vague CTAs. "Follow for more" is dead. The CTA must specify **the action**, **the value of taking it**, and **the friction level**.

### CTA patterns that work

1. **The save** — *Save this so you have it when you need it.* (Specific value, zero friction.)
2. **The share** — *Send this to the [specific person] who needs it.* (Friction = naming someone, but value is implicit social currency.)
3. **The comment** — *Comment "[word]" and I'll DM you [resource].* (Specific friction, specific reward.)
4. **The link** — *Full breakdown linked in bio.* (Only use when the link genuinely extends the carousel.)
5. **The follow** — *Follow [@handle] for [specific recurring value], every [cadence].* (Never just "follow for more.")

### CTA slide layout

- Black background
- Teal accent: a horizontal bar, button shape, or large number
- White CTA text, 72pt, ≤ 12 words
- Handle bottom-left in white
- Page indicator top-right in teal (still required)

## Voice rules

- **Active voice.** "Posting works" not "Posts are made by posting."
- **Second person.** Address the reader as "you" — never "one" or "the user."
- **Specific over abstract.** Numbers, names, examples beat generalities.
- **No emoji in body copy.** Optional emoji in the CTA only, max one.
- **Sentence case for all headlines, including the cover.** Title Case feels corporate; ALL CAPS undermines Fraunces' editorial authority (the high-contrast serif is doing the work — shouting on top of it reads as trying too hard). Sentence case lets the typeface command attention without competing with the words.

## Editing pass (do this before generating visuals)

1. Read every slide aloud. If you stumble, cut.
2. For every slide, ask: *would the reader swipe to the next slide after this?* If no, rewrite or merge.
3. For the cover: would you swipe? If you wouldn't, neither will anyone else.
4. Check word counts: cover ≤ 8 words, value slides ≤ 50 words, CTA ≤ 12 words.
5. Check that the last slide's CTA names a specific action and reward.

## When the user gives raw copy

If the user pastes existing copy and asks to "make a carousel from this":

1. Identify the single core promise — usually the first or last sentence.
2. Make that the cover hook.
3. Extract 3-5 distinct ideas from the body. Each becomes one value slide.
4. If the source has no CTA, ask the user what they want the reader to do next.
5. Show the user the slide-by-slide breakdown before generating.
